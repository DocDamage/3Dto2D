# SpriteForge Studio v12 — Final Polish Edition

Welcome to **SpriteForge Studio**, a professional, local pipeline designed to generate game-ready, transparent, looping 2D sprite sheets directly from text prompts, reference images, or existing video files. 

By wrapping ComfyUI and WAN (Hunyuan Latent Video) models into an operational desktop dashboard, SpriteForge automates video rendering, background removal (chroma keying), anchor stabilization, loop-seam stitching, and engine-ready exports.

## 🚀 Quick Start Guide

1. Double-click **`START_HERE.bat`** on Windows, **`start.command`** on macOS, or run **`./start.sh`** on Linux (or execute `app/spriteforge_launcher.py` in your terminal).
2. The local web server will launch and automatically open the SpriteForge Dashboard in your web browser.
3. **Run the No-GPU Demo:** Go to the **Launchpad** tab and click **Run No-GPU Demo** to verify that Pillow, FFmpeg, and chroma-key utilities are functional on your system.
4. **Install Safe WAN Setup:** Go to the **Setup** tab and click **Install Everything: Safe Wan 2.1**. This sets up ComfyUI, custom nodes, and downloads the default 9.8 GB Wan 2.1 T2V 1.3B models.
5. **Launch ComfyUI:** Go to the **Setup** tab and click **Launch ComfyUI** (or run `START_HERE.bat --comfy`).
6. **Make Sprite:** Go to the **Make Sprite** tab and generate your first character sprites! Use the `debug` profile first to verify, and then `rtx3060_12gb` or higher.
7. **QA & Auto-Fix:** Go to the **Quality Lab** tab to run QA checks or apply automated fixes (e.g., stabilizing feet or fixing seams).
8. **Build Release:** Go to the **Release** tab to bundle your finished animations into a release ZIP.

## 💻 Hardware Modes (GPU vs. CPU)

- **🟢 Local GPU Mode (RTX 3060+ with 12GB+ VRAM):** Required for generating new sprites locally. Automatically scales down resolution and switches to safe profiles if a CUDA Out Of Memory (OOM) error occurs.
- **🟡 No-GPU Mode / CPU Mode (Demo & Video Conversion):** You do not need a high-end GPU to run SpriteForge's processing, packing, or QA tools. You can drop any existing `.mp4`/`.webm` video file in the **Convert Video** tab to remove the background, feather edges, and slice it into a sheet locally.

## 🔄 Advanced Features in v12 Final Polish

1. **In-Browser Frame Painting:** Fix chroma halos, erase artifacts, and touch up frames using the drawing board directly inside the Quality Lab.
2. **Color Palette Locking:** Snap final frames to retro palettes (PICO-8, NES, Gameboy, or custom hex lists) using the `--palette` option or the post-processing panel.
3. **Collaborative Project Sharing:** Export and import portable `.spriteforge` ZIP bundles directly from the Projects page to share character assets.
4. **Video Pose Estimation:** Run MediaPipe rotoscoping on reference videos to extract OpenPose-style skeleton guides.
5. **Tilemap & Environment Sprites:** Use presets for animated liquid surface tiles (water, lava) and autotiling compilation helpers.
6. **Plugin / Extension System:** Hook custom QA metrics (`on_qa_check`) or engine exports (`on_export_engine`) using Python scripts under the `app/plugins/` directory.
